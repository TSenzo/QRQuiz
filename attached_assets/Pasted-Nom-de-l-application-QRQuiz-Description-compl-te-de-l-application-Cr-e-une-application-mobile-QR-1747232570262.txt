Nom de l'application : QRQuiz
Description complète de l'application :
Crée une application mobile QRQuiz avec les fonctionnalités suivantes :

Création de quiz :

Permet à l'utilisateur de créer un quiz avec plusieurs questions à choix multiples, chaque question ayant des réponses possibles (texte ou image).

L'utilisateur peut définir la bonne réponse pour chaque question.

Ajouter des images ou vidéos comme contenu de quiz pour chaque question.

Générer un QR code pour partager le quiz avec d'autres utilisateurs, où le lien vers le quiz est inclus dans le QR code.

Stocker localement les quiz créés avec des données simples (question, options, bonne réponse).

Scanner un QR code :

L'utilisateur peut scanner un QR code contenant un lien vers un quiz.

Dès que le QR code est scanné, l'application charge le quiz et l'affiche à l'utilisateur pour qu'il puisse y répondre.

Si le QR code est invalide ou corrompu, l'application doit afficher un message d'erreur convivial (ex. "QR code invalide" ou "Quiz non trouvé").

Répondre au quiz et voir les résultats :

L'utilisateur peut répondre aux questions d'un quiz via des boutons de sélection.

Après avoir répondu à toutes les questions, l'application affiche les résultats : le score total et les réponses correctes/incorrectes.

L'utilisateur peut visualiser les bonnes réponses après avoir soumis ses réponses.

Classement local :

L'application garde une trace des meilleurs scores des utilisateurs dans un classement local.

Les scores sont stockés dans une base de données locale et sont triés par ordre décroissant.

Option de réinitialiser ou supprimer les scores si nécessaire.

Publicités et monétisation :

Ajouter des publicités AdMob pour générer des revenus (bannières et vidéos).

Ajouter des achats in-app pour débloquer des fonctionnalités premium, comme des options avancées pour personnaliser le quiz, plus de quiz à créer, ou des thèmes premium pour les quiz.

L'intégration des achats in-app doit permettre un flux simple pour l'utilisateur via Google Play Store.

Fonctionnement hors-ligne :

Toute l'application doit fonctionner entièrement hors-ligne, sans nécessiter de connexion à Internet, en utilisant une base de données locale pour stocker les quiz, les résultats, et le classement.

Lorsqu'il n'y a pas de connexion Internet, l'application doit continuer à fonctionner avec toutes les fonctionnalités accessibles.

Interface utilisateur (UI/UX) :

Design simple, moderne et épuré.

Interface claire pour naviguer entre les fonctionnalités : créer un quiz, scanner un QR code, consulter les résultats, et voir le classement.

Utilisation d'animations pour une transition fluide entre les écrans (par exemple, transition entre les questions du quiz).

Les boutons et éléments doivent être facilement cliquables et bien espacés.

Utiliser des couleurs vives et contrastées pour une expérience utilisateur agréable.

Optimisation et performance :

Assurer des temps de réponse rapides, même avec des quiz contenant plusieurs questions et réponses.

Optimiser l'usage de la mémoire et de la batterie pour une utilisation prolongée sans ralentissement.

L'application doit être compatible avec toutes les tailles d'écran mobile et les résolutions courantes.

Gestion des erreurs :

Afficher des messages d'erreur clairs et utiles en cas de problème (par exemple, si l'utilisateur n'a pas créé de quiz ou si un quiz ne se charge pas).

Gérer les erreurs liées à l'usage des QR codes (QR non valide, erreur de connexion pour les achats in-app, etc.).

Sécurité et confidentialité :

Assurer que toutes les données des utilisateurs sont stockées de manière sécurisée et que la confidentialité est respectée.

L'application ne doit pas collecter ou partager d'informations sensibles sans le consentement explicite de l'utilisateur.

Technologies à utiliser (si nécessaire pour l'IA) :

Base de données locale : Utiliser Room ou une autre base de données locale pour stocker les quiz, résultats et scores.

QR code : Génération et scan de QR codes via des bibliothèques comme ZXing.

Publicités : Intégration d'AdMob pour afficher des publicités (bannières, interstitielles, vidéos récompensées).

Achats in-app : Implémentation via Google Play Billing pour offrir des fonctionnalités premium.

Hors-ligne : Toutes les données doivent être stockées localement, permettant un fonctionnement complet sans Internet.
